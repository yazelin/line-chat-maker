# 隱形識別標記:原理與界限

匯出的 PNG 一律嵌入三層識別標記(無開關),任何人可用 [verify.html](../verify.html) 在瀏覽器本機驗證「這張圖是否由本工具產生」。本文記錄設計原理、實測數據與誠實界限。

## 威脅模型

目的不是防偽,是**提高順手濫用的成本**,並給受害者與平台一個識別線索。本專案開源,有心人 fork 掉幾行程式碼就能移除標記;別的工具做的假圖這裡也驗不出來。「未檢出」永遠不等於「圖是真的」。

## 三層架構

| 層 | 機制 | 存活範圍 |
|---|---|---|
| 檔案 metadata | PNG iTXt chunk(插在 IHDR 後),UTF-8 識別字串 | 檔案原樣流傳;上傳社群平台通常被移除 |
| 像素藏碼 | alpha 通道 LSB,2 bytes 長度 + UTF-8 字串 | 未轉檔、未重壓縮的原始 PNG |
| 隱形場紋 | 藍通道 ±2/255 平滑場(下述) | 截圖、JPEG 重壓縮(實測 q70 以上)、裁切;縮放後不保證 |

技術選擇備註:metadata 用 iTXt 而非 tEXt,因為 tEXt 只吃 Latin-1;藏碼放 alpha 通道而非 RGB,因為 canvas 是 premultiplied alpha,半透明像素的 RGB 會在讀寫間失真,alpha 本身不會。

## 場紋嵌入

- **圖樣**:16px 網格節點值 ±1(座標雜湊,`Math.imul` 可跨平台位元級重現),smoothstep 雙線性內插成平滑場,週期 8 節點(128px)。不用文字遮罩,因為字型渲染跨平台不一致,驗證端無法重現。
- **通道**:只動藍色(人眼對藍的空間解析度最低),振幅 ±2。
- **抖動**:量化前加 4×4 Bayer 有序抖動。直接 round 會產生量化等高線,在平坦底色上是可見斑塊;抖動把它打散成高頻微噪,而 16px 區塊均值不變,偵測不受影響。
- **近白/近黑**:振幅減半,截波後只剩單邊(白上為 0 或 −1)。實測 −2 的平滑斑在純白上可見(偏黃色調),−1 是量化底線。

### 可見性迭代教訓(2026-07-17)

1. 硬邊 ±3 亮度格子:平坦底色上是規律 banding,振幅再小人眼也抓得到結構性邊緣。使用者實測可見,棄用。
2. 平滑場 ±3 藍通道:平坦底色上仍有可察覺的雲狀斑。棄用。
3. round 量化等高線:平滑場被取整後留下軟邊斑,加 Bayer 抖動解決。
4. 白色補償平移(把 ±2 映成 −4..0):統計漂亮,但白上是明顯黃斑,棄用。守則:**眼睛驗證優先於統計數字**。

## 偵測(verify.html)

- **觀測值**:每像素取 B−(R+G)/2。聊天畫面大多中性色,這個差值幾乎為零,雜訊底遠低於亮度;場紋只動藍通道,訊號完整保留。
- **積分圖**:一次掃描建 值/平方/不透明數 三張積分圖,之後任意 16px 區塊的均值與變異數 O(1) 可查。
- **對齊搜尋**:裁切會移動格線。搜尋 16×16 像素相位 × 8×8 圖樣平移 = 16384 個候選,每個候選做匹配濾波。
- **模板**:區塊均值的期望值恰好是四角節點平均(smoothstep 積分對稱),8×8 torus 預先算好高通版。
- **高通與截尾**:區塊均值減四鄰均值壓掉影像內容;彩色元素邊緣是重尾 outlier(綠泡泡邊可達 ±100),clip 在 3σ 內,±1 級的訊號不受影響。
- **統計量**:正規化匹配濾波 t 值,null 近似 N(0,1)。
- **平坦鄰域子集**:另算一組只含「中心與四鄰全平坦」區塊的 t 值。白底圖訊號被截波剩半幅,但平坦區塊雜訊近零,SNR 反而更高。鄰居必須也平坦,否則鄰居的內容會經高通(減四鄰)漏進來淹沒訊號——這是實測踩過的坑。σ 設下限 0.3,防止乾淨平坦圖 σ→0 讓 t 爆衝成誤判。
- **門檻**:z > 7 判檢出。null 是約 3 萬個 N(0,1) 的極值,期望約 4.4;實測乾淨圖與真實照片 z ≤ 4.1。

## 已知界限

- 開源:fork 移除 `brandPixels`/`pngWithMeta` 即可拿掉標記。
- 縮放(含非 100% 的截圖)會破壞 16px 格線對齊,場紋不保證驗出。
- JPEG 品質低於 q70 未測試,不承諾。
- 對外文案守則:只說「提高濫用成本、可供識別」,不說「消不掉」。

## 重測方式

用 Playwright 走真實「匯出 PNG」按鈕與真實 verify.html 頁面(setInputFiles),對匯出檔做 JPEG 重壓縮與裁切後再驗,並跑乾淨圖與無關照片的誤判組。注意:測試 context 要 `serviceWorkers: 'block'`,否則 app 的 service worker 會以 `ignoreSearch` 快取測試圖片,不同變體會拿到同一張圖(實測踩過)。
